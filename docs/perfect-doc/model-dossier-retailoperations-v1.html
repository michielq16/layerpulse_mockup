⚠ ROOT CAUSE (explains §06): all 12 source tables read from a local desktop path (C:\Users\michi\…). The Fabric service has no line of sight to a workstation's file system, so every scheduled refresh fails by design. Fix = repoint to a service-reachable source (Lakehouse, OneLake, SharePoint, or blob) and the model lights up immediately — nothing else about it is broken.
